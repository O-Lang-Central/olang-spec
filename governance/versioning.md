# Versioning Policy

O-Lang uses **semantic versioning** for the protocol:

- **MAJOR** (`2.0`): Breaking changes to normative behavior
- **MINOR** (`1.2`): New features, non-breaking
- **PATCH** (`1.1.1`): Reserved for errata only (no behavioral changes)

Conformance is **strictly version-bound**. A v1.1-conformant resolver is not automatically v1.2-conformant.
